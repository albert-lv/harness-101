从零构建 AI Agent 与 Harness Engineering 实战课程

一门面向工程师的动手课：先手写一个 Agent，再把它打磨成可靠的运行时（Harness）。不依赖重型框架，每一章都有可运行产出物。

不要把 Agent 当 AI 来学，把它当成一个不可靠的分布式系统来学。

前置要求：Python 基础、命令行、一个 LLM API Key（推荐 Anthropic Claude，亦可用兼容 API）。

模块 0：环境与第一次 API 调用

学习目标：搭好环境，理解一次 LLM 请求/响应最小结构。

核心内容

  • SDK 安装、API Key 环境变量配置。
  • 最小调用三要素：system prompt、messages、模型参数。
  • 理解 token 计费与长度约束。
  • 流式与非流式响应差异。

动手练习

  1. 写 chat.py，命令行输入一句话并打印回复。
  2. 加多轮循环并维护 messages，观察 context 增长。
产出物：可多轮对话并打印 token 消耗的命令行脚本。

模块 1：手写 Agent Loop

学习目标：实现带工具的 REPL 循环，理解 Agent 本质是状态机。

核心内容

  • 调模型 → 解析工具调用 → 执行工具 → 结果回填 context 直到完成。
  • Tool calling 协议（name/description/参数 schema）。
  • 解析 tool_use 并映射到 Python 函数。
  • 模型不再请求工具时终止循环。

动手练习

  1. 不用框架，约 100 行实现 loop。
  2. 注册第一个工具（如 run_command / 文件读取 / 计算器）。
  3. 完成 2-3 步工具调用任务并打印每一步动作。
产出物：最小但完整的多步 Agent。

模块 2：工具设计

学习目标：设计让模型用得对、用得稳的工具接口。

核心内容

  • 工具 description 本身就是 prompt。
  • 参数校验必须在工具内进行。
  • 返回值要利于模型继续决策。
  • 错误作为正常返回值，避免 loop 崩溃。
  • 有副作用工具要考虑幂等。

动手练习

  1. 重构模块 1 工具，加参数校验和友好错误。
  2. 设计有副作用工具并防止重复执行。
  3. 故意写模糊 description，观察误用。
产出物：一套健壮工具规范 + 2-3 个示例工具。

模块 3：Context 管理

学习目标：处理 context 爆掉问题。

核心内容

  • 长任务和大工具输出会快速撑满窗口。
  • 策略：截断、摘要、外置记忆。
  • 在信息损失和成本之间做权衡。
  • 区分常驻 system prompt 与动态注入信息。

动手练习

  1. 用超长输出工具复现 context 爆掉。
  2. 实现截断与阈值摘要策略。
  3. 比较两种策略对任务质量影响。
产出物：长任务下仍稳定的 Agent。

模块 4：失败模式与容错（Harness 入门）★

学习目标：系统性修复死循环、跑偏、幻觉调用与部分失败。

核心内容

  • 死循环检测与打断（最大步数、重复检测）。
  • 阶段目标与自检避免跑偏。
  • 拦截未知工具与异常参数。
  • 多步任务的部分失败恢复。
  • 限流/超时重试与退避。

动手练习

  1. 人为注入工具错误、限流、未知工具。
  2. 为每种失败加 Harness 防护。
  3. 记录真实任务跑偏位置并改进。
产出物：能从多种错误中恢复的 Agent。

模块 5：权限与沙箱

学习目标：让 Agent 安全地访问真实世界。

核心内容

  • 工具白名单与上下文级权限控制。
  • 危险操作需人工确认（挂起-恢复）。
  • 沙箱执行：文件系统和网络隔离。
  • 最小权限原则：默认只读，显式授权写入。

动手练习

  1. 给副作用工具增加确认门。
  2. 实现只读模式拦截写操作。
  3. 编写权限模型清单。
产出物：默认最小权限、危险操作需确认的 Agent。

模块 6：可观测性——Trace 与 Events

学习目标：让每次运行可见、可回放、可 debug。

核心内容

  • 结构化事件：思考、工具调用、结果、错误、完成。
  • 结构化日志：输入输出、token、耗时。
  • 基于 trace 复现与排障。
  • 成本观测：token、时延、费用。

动手练习

  1. 输出 JSONL 事件日志。
  2. 实现一个简易 trace 时间线查看器。
  3. 用 trace 定位一次跑偏根因。
产出物：每次运行都可追踪回放的 Agent。

模块 7：多 Agent 编排

学习目标：从单 Agent 升级到主-子编排器。

核心内容

  • 适用场景：并行分解、上下文隔离、角色分工。
  • orchestrator-worker 模式：拆分、派发、汇总。
  • 并行执行与结果合并策略。
  • 子 Agent 失败隔离与恢复。

动手练习

  1. 实现主 Agent 拆分子任务。
  2. 并行运行多个子 Agent 并汇总结果。
  3. 复用模块 6 trace 支持编排调试。
产出物：可并行执行并汇总的多 Agent 编排器。

模块 8：阅读真实 Harness

学习目标：带着问题读成熟 Harness 源码并迁移改进。

核心内容

  • 重点看 context 管理、容错、权限、可观测、编排。
  • 按问题驱动阅读，不盲目通读。
  • 抽取可迁移设计并落地到自己的 Agent。

动手练习

  1. 围绕 5 个问题各写一段精读笔记。
  2. 迁移至少 1 处设计到自己的 Agent。
产出物：源码精读笔记 + 至少一项工程改进。

结课项目：一个完整的领域 Agent

目标：综合所有模块，交付一个真实可用的领域 Agent。
  • 运维/网络方向：并行排查多设备，写操作需确认，全程 trace。
  • 数据方向：自主查询、清洗、生成报告。
  • 开发方向：读代码、定位 bug、提出修复。

最低合格检查清单

  • 手写 agent loop（非重型框架）
  • 有校验且错误优雅返回的工具
  • context 管理（长任务不崩）
  • 失败恢复（工具错误、死循环、限流）
  • 权限模型（确认门 + 最小权限）
  • 完整 trace（可回放、可 debug）
  • 进阶：多 agent 编排
先让它工作，再让它可靠，最后才让它复杂。
