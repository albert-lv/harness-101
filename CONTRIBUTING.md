# Contributing to Agent Harness 101

Thank you for helping improve this course!

## Content Workflow

1. Edit `agent-harness-course.md` as the source of truth.
2. Sync the updated content to `index.html`.
3. If you add or change terminology, update `GLOSSARY.md`.
4. If you add code examples, place them under `examples/prep/` or `examples/factor-N/{starter,solution}/` and update `examples/README.md`.
5. Open a Pull Request with a clear summary of what changed and why.

## What to Contribute

- Typo fixes and clarifications
- New diagrams or code examples
- Suggestions for additional exercises
- Translations (please coordinate via an issue first)
- Improvements to the **Harness 12-Factor** checklist

## Style Guidelines

- Keep explanations beginner-friendly.
- Provide runnable code snippets where possible.
- Use consistent terminology (see [`GLOSSARY.md`](GLOSSARY.md)).
- Prefer concise examples over long prose.
- Keep `index.html` a zero-dependency single file; new pages should be discussed in an issue first.
